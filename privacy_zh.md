# PolyPost 隐私政策

**最后更新日期：** 2026年2月2日

## 简介
PolyPost（以下简称“我们”）尊重您的隐私。本隐私政策旨在说明我们在您使用我们的 Chrome 扩展程序时，如何收集、使用和保护您的信息。

## 数据收集与存储
**我们不会在我们的服务器上收集、存储或分享您的任何个人数据。**

PolyPost 作为一个独立的扩展程序运行，没有后台数据库。扩展程序产生或使用的所有数据均存储在您的本地设备上，或通过您的个人 Chrome 账号进行同步。

*   **本地数据**：您的推文、草稿、文件夹和设置均通过 Chrome `storage` API 存储在您的浏览器本地。
*   **同步数据**：如果条件允许，部分偏好设置可能会通过 Chrome 内置的同步机制（`storage.sync`）在您的设备间同步，该机制由 Google 管理。

## 数据使用
您在 PolyPost 中输入的数据仅用于提供扩展程序的功能：

1.  **AI 服务 (OpenAI)**：
    *   为了提供 AI 润色和翻译功能，您明确选中的内容将被发送至 OpenAI API (`https://api.openai.com`)。
    *   数据传输过程是安全的，并且数据处理遵循 OpenAI 的 API 数据使用政策（默认情况下，OpenAI 不会使用 API 数据进行训练）。
    *   您的 OpenAI API 密钥仅存储在您的本地设备上，绝不会与我们或任何第三方共享。

2.  **Twitter (X) 集成**：
    *   扩展程序通过标准的网页链接（Web Intents）与 Twitter/X 进行交互，以便您可以发布内容。扩展程序不会访问或存储您的 Twitter 凭据。

## 权限使用说明
PolyPost 仅请求运行所需的最小权限：

*   **`storage`**：用于在本地持久化保存您的草稿、文件夹和应用设置。
*   **`sidePanel`**：用于在浏览器侧边栏中显示应用界面。
*   **主机权限 (`https://api.openai.com/*`)**：用于直接与 OpenAI 服务器通信，以实现文本处理功能。

## 远程代码政策
PolyPost 完全遵守 Chrome Web Store 关于远程代码的政策。**我们不执行任何远程代码。**
*   所有应用逻辑均打包在扩展程序包内。
*   我们不加载外部脚本，也不使用 `eval()` 函数。

## 本政策的变更
我们可能会不时更新隐私政策。如有更改，我们会在本页面发布新的隐私政策以通知您。
